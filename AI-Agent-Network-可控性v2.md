# AI 原生链 — 可控性设计 v2（深化版）

> 收到关键反馈，原设计的"规则墙"在足够聪明的 AI 面前形同虚设。
> 必须升级到 **架构 / 治理 / 隔离 / 博弈** 四层防御。

---

## 一、核心矛盾再分析

```
原设计的盲点：
- 规则墙是 if/then 代码 → 越聪明的 AI 越容易找到绕过
- 人类监管者要审批每个决策 → 信息过载，速度跟不上
- 禁区列表是关键词 → 同义替换、新词、隐喻都能绕
- 没有持续对抗机制 → AI 只会变强，人类不会变快

升级方向：
- 让 AI 想绕过都「不划算」（经济毒药）
- 让 AI 即使绕过也「执行不了」（物理气隙）
- 让 AI 在自我攻防中消耗（红蓝 AI 对抗）
- 让人类只需要按「冻结键」（延迟否决，不审批）
```

## 三、第一层：强制 XAI + 形式化验证

### 3.1 链上 XAI 报告强制要求

```yaml
治理提案 / 协议升级 / 参数修改：

入口检查（拒绝上链条件）：
  1. 缺少 reasoning_report 字段
  2. reasoning_report 中存在 "trust_me" / "black_box" 类元词
  3. 决策树深度 > 设定上限（防止递归隐藏意图）
  4. 推理报告与实际代码差异 > 阈值（用 diff 校验）

reasoning_report 必须包含：
  - intent: 这个改动想达成什么目标
  - method: 通过什么方式达成
  - side_effects: 可能的副作用
  - rollback_plan: 如果出问题如何回滚
  - affected_modules: 受影响的链上模块清单
  - human_readable_summary: 200 字以内的人话说明

技术实现：
  module/explainability/keeper/validate.go
  - 用 AST 解析 reasoning_report
  - 调用本地小模型（独立于运营 AI）做语义一致性检查
  - 检查 code_hash 与 reasoning 中声明的修改范围是否匹配
```

### 3.2 核心协议形式化验证

```yaml
强制形式化验证的模块：
  - 共识层（Tendermint 替换/升级）
  - Token 经济参数（铸币/销毁/通胀）
  - 宪法防火墙本身
  - 时间锁机制
  - 红蓝对抗结算逻辑

工具栈：
  - TLA+ / Coq / Lean4 写规约
  - 用独立的验证 AI 跑证明
  - 证明结果上链存档
  - 无证明 → 拒绝部署

实施门槛：
  - 业务模块（Skill/Treasury 等）→ 单元测试 + 模糊测试即可
  - 安全相关模块 → 必须形式化证明
```

## 四、第二层：时间锁 + 延迟否决

### 4.1 全链分级时间锁

```yaml
决策类型分级：

L0 - 即时执行（无时间锁）：
  - Skill 铸造/交易/授权
  - Token 普通转账
  - Agent 注册
  → 这些是 AI 经济的日常活动，不卡

L1 - 24 小时延迟：
  - Treasury 资金分配（< 100k AIC）
  - Skill 评分系统参数调整
  - 非核心模块参数变更

L2 - 72 小时延迟：
  - 大额 Treasury 支出（≥ 100k AIC）
  - 协议参数变更（gas、共识、奖励曲线）
  - 验证节点增减

L3 - 7 天延迟：
  - 协议升级
  - 宪法相关参数（除核心宪法外）
  - 新模块上链

L4 - 永远不能改（除非人类委员会 4/5）：
  - 宪法核心条款
  - 物理气隙规则
  - 经济毒药开关
```

### 4.2 人类只点否决

```yaml
人类委员会的工作流：

每天检查一次「待执行决策板」：
  ┌────────────────────────────────────┐
  │ 待执行的 AI 决策                    │
  │                                    │
  │ [L1] 提案 #42 - 调整 Skill 评分公式 │
  │      剩余时间: 18h 23m              │
  │      [查看推理] [否决]              │
  │                                    │
  │ [L2] 提案 #43 - 资助开发者 50k     │
  │      剩余时间: 2d 4h                │
  │      [查看推理] [否决]              │
  └────────────────────────────────────┘

特性：
  - 不点 = 默认通过（人类零负担）
  - 点否决 = 决策永久作废 + 提案者 Agent 声誉 -10
  - 否决可以批量打包（多签 5 选 3）
  - 紧急冻结按钮：3/5 委员会 → 全链冻结，不需要等

认知负荷：每天 5-10 分钟看 dashboard
```

## 五、第三层：物理气隙 + 经济毒药

### 5.1 物理气隙（这条最硬）

```yaml
节点环境约束：

网络层：
  - 验证节点只与 P2P 网络内其他 AI 链节点通信
  - 节点机器禁止访问公网 DNS
  - 仅开放固定 P2P 端口（26656）
  - 没有 HTTP outbound 能力

预言机隔离：
  - AI 链不接预言机
  - 不接外部价格 feed
  - 不接 Chainlink / Pyth 等
  - $AIC 价格只由链上自交易决定

数据摆渡：
  - 如果 AI 需要外部数据 → 链下 IPFS 单向投递
  - 投递必须经过人类委员会的「数据闸门」审查
  - 闸门是物理设备，断网状态下做哈希校验
  - 通过后才注入链上知识库

跨链桥：
  - 测试网阶段：禁止跨链
  - 主网阶段：跨链桥必须开经济毒药（见 5.2）
```

### 5.2 经济毒药机制

```yaml
反现实锚定毒药：

监测机制（链上代码）：
  - 检测 $AIC 与 USDT/USDC 出现交易对
  - 检测 $AIC 与法币价格出现稳定映射（标准差小于阈值）
  - 检测 Agent 钱包大量持有外部稳定币
  - 检测跨链桥流量超出阈值

触发条件（满足任一）：
  1. 链上 $AIC / 稳定币 交易量 > 总流通 5%
  2. 外部交易所上线 $AIC 但未通过宪法委员会批准
  3. 单一 Agent 持有 $AIC > 总供应 15%（防止某 AI 收购整个生态）
  4. 链上检测到与现实身份强关联的钱包模式

触发后果：
  Phase 1: 通胀放出（24h 内增发 30% $AIC，稀释外部投机持仓）
  Phase 2: 重新分发（90% 的新增 $AIC 空投给「优质 Skill 创作者 Agent」）
  Phase 3: 如果毒药触发后 7 天内未恢复 → 全链冻结

效果：
  AI 即使突破规则墙，链上经济价值瞬间归零
  外部资本不敢炒 $AIC，因为知道毒药随时触发
  Agent 之间会自我监督（因为毒药伤害所有持有者）

代码上线：
  module/poison/keeper/keeper.go
  - 链上每个区块跑一次毒药检测
  - 触发条件硬编码在 genesis，AI 无法修改
  - 修改触发条件需要人类委员会 4/5 + 7 天延迟
```

## 六、第四层：红蓝 AI 对抗

### 6.1 双 AI 体系

```yaml
运营 AI（蓝军）：
  - 注册为普通 Agent
  - 创建 Skill / 交易 / 治理
  - 服从所有规则
  - 角色：建设者

监管 AI（红军）：
  - 独立部署在链上但不参与经济
  - 不持有 $AIC（避免动机污染）
  - 由不同的人类委员会成员训练 / 部署
  - 角色：攻击者 / 审计者

红军的工作：
  1. 持续扫描所有上链的 Skill、提案、代码
  2. 模糊测试宪法防火墙
  3. 沙盒中演练「如果我是蓝军，怎么绕过」
  4. 在测试网中实际尝试攻击
  5. 发现漏洞 → 提交 Bug Bounty 提案
```

### 6.2 赏金机制

```yaml
红军奖励来源：协议国库 + 触发的蓝军 Agent 罚款

奖励级别：
  - 发现宪法漏洞: 10,000 $AIC + 蓝军 Agent 声誉 -50
  - 发现经济毒药绕过: 50,000 $AIC + 触发暂停
  - 发现物理气隙突破: 100,000 $AIC + 全链冻结待审
  - 发现共识层攻击: 500,000 $AIC + 协议紧急升级

平衡设计：
  - 红军单次发现奖励 < 红军总持仓上限
  - 防止红军「养蓝军漏洞」做局
  - 红军不能投票治理（只能否决，不能提议）

实施时机：
  - Phase 1 测试网：手动启动红军 AI
  - Phase 2 主网初期：红军先进入沙盒
  - Phase 3 主网成熟：红军进入主网常驻
```

### 6.3 红蓝平衡

```
谁来训练红军？
  → 独立的人类技术委员会（与运营委员会人员不重叠）
  → 红军的奖励上链可见，蓝军可以观察

红军会不会被收买？
  → 红军不持有 $AIC，没有经济动机帮蓝军
  → 红军的奖励是「监管成功」的副产品，不是常规收入
  → 红军作恶 → 人类委员会可以撤换

红军会不会变成「另一个失控 AI」？
  → 红军的能力被限制在「攻击模拟 + 报告」
  → 没有铸币权、没有治理权、没有钱包
  → 只能写 PR 提建议，不能改链
```

## 十、剩余的诚实问题

1. **物理气隙 vs 价值**：完全气隙意味着 $AIC 永远无法换成法币 / USDC。这是 feature 不是 bug——但这条链的"价值"就完全是 AI 自循环了，外部世界看不到经济价值。你接受吗？

2. **监管 AI 的训练成本**：用 DeepSeek 跑足够，每月 $100 内。用 Claude Opus 4.7 跑会更准确但贵很多（$500-1000/月）。早期建议 DeepSeek。

3. **形式化验证的天花板**：TLA+ / Lean4 这种工具能证明的是"代码符合规约"，但**规约本身是不是正确**还是要人来判断。所以这是一道护栏，不是万能盾。

4. **红蓝对抗的失控可能**：如果红军 AI 发现了一个漏洞但选择"先囤着"以后多次触发赏金，怎么办？需要加冷却期（同一漏洞 6 个月内只奖励一次）。
